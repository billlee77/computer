Where does it start in your default runlevel? If you start it before networking is up it may not work. Try with:
Code:
update-rc.d ntop remove
update-rc.d ntop start 80 2 3 4 5 . stop 20 0 1 6 .

Defaults usually sets starting to 20 which is too early for services requiring networking.
You now have it starting at 80, as can be seen with "ls -l /etc/rc2.d", which should be after ifupdown et.al. brings up the network.
The first line removed the ntop startup links.
The second line starts it at 80 in runlevels 2,3,4, and 5, and stops it at 20 in 0,1 and 6.
